The {url-javadoc}{swing-framework-ui}/is/codion/swing/framework/ui/EntityApplicationPanel.html[EntityApplicationPanel] class serves as the main application UI.
When extending this class you must provide a constructor with a single application model parameter, as seen below.

The constructor takes the application's root entity panels, and optionally a set of *lookup panel* builders: panels for supporting entities — lookup and reference data — which appear in the application's *View* menu and open on demand, in their own windows, rather than occupying a tab.

== Starting the application

An application is assembled and started with the {url-javadoc}{swing-framework-ui}/is/codion/swing/framework/ui/EntityApplication.html[EntityApplication] builder, which handles the startup sequence: look and feel, login (unless a user is provided), connection, application model and panel construction, and the main frame.
The *main* method is also the natural place to configure framework defaults — configuration values apply to every panel created after them.

== Layout

The root entity panels are laid out by an {url-javadoc}{swing-framework-ui}/is/codion/swing/framework/ui/EntityApplicationPanel.ApplicationLayout.html[ApplicationLayout] — by default a {url-javadoc}{swing-framework-ui}/is/codion/swing/framework/ui/TabbedApplicationLayout.html[TabbedApplicationLayout], one tab per root panel, initialized lazily as they are first displayed.
Supply a custom layout via the *EntityApplicationPanel* constructor to arrange the root panels differently.

== SQL Tracing

NOTE: SQL Tracing is only available when using a local EntityConnection.

Application *SQL Tracing* can be enabled via the {url-javadoc}{swing-framework-ui}/is/codion/swing/framework/ui/EntityApplicationPanel.html#SQL_TRACING[EntityApplicationPanel.SQL_TRACING] configuration value.

This configures the underlying EntityConnection to trace its queries and adds a *SQL Trace* item under *Help* -> *Log* main menu, with actions to enable tracing and view the trace log.

Disabling SQL Tracing clears the trace log.

[source,java]
----
EntityApplicationPanel.SQL_TRACING.set(true);
----
