= Framework Model Architecture
:dir-chinook-source: ../../../../../demos/chinook/src/main/java
:url-javadoc: link:../api

The model layer is a complete, UI-independent application: data retrieval, editing, validation, selection, master-detail coordination and all associated state live here, exposed through the framework's <<_reactive_classes, reactive classes>>.
The UI layer *renders* models — it does not extend them with logic.
Nothing in the model layer depends on a UI toolkit, which has two practical consequences: application logic is unit-testable without showing a window, and the same models can drive different client technologies.
