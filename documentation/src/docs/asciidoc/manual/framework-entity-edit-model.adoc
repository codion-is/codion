EntityEditModel --o EntityEditor
-------------------------------------------

The {url-javadoc}{framework-model}/is/codion/framework/model/EntityEditModel.html[EntityEditModel] binds an <<_entityeditor, EntityEditor>> to a connection, and is the model an {url-javadoc}{swing-framework-ui}/is/codion/swing/framework/ui/EntityEditPanel.html[EntityEditPanel] is based on.
The editing itself — values, validation, dirty state, insert, update and delete — is the {url-javadoc}{framework-model}/is/codion/framework/model/EntityEditor.html[EntityEditor]'s job, accessed via {url-javadoc}{framework-model}/is/codion/framework/model/EntityEditModel.html#editor()[editor()].

== Combo box models

The Swing implementation, {url-javadoc}{swing-framework-model}/is/codion/swing/framework/model/SwingEntityEditModel.html[SwingEntityEditModel], provides combo box models for foreign keys and column values, shared by the input components bound to them.
A combo box model can be initialized eagerly in the constructor — otherwise it is created and refreshed on first use, by the component requesting it.

[source,java,indent=0]
----
include::{dir-chinook-source}/is/codion/demos/chinook/model/TrackEditModel.java[tags=trackEditModel]
----

Combo box models based on entities stay consistent automatically: entities inserted, updated or deleted elsewhere in the application are added to, replaced in or removed from the combo box model, via the editor's insert, update and delete events.
